# selection-sort-php

Учебная реализация сортировки выбором для PHP 8.1+.

Проект показывает:

- последовательный поиск минимального элемента в неотсортированной части массива;
- сортировку обменом без удаления элементов через `array_splice()`;
- пользовательский порядок через callback-компаратор;
- сортировку объектов и сложных структур;
- автоматические тесты на PHP 8.1–8.4.

> Для прикладного production-кода обычно следует использовать встроенные `sort()`, `usort()` и другие функции PHP. Они существенно быстрее на больших наборах данных. Эта реализация предназначена для изучения алгоритма.

## Установка

```bash
composer require da41b94c/selection-sort-php
```

До публикации пакета в Packagist репозиторий можно подключить через секцию `repositories` Composer или клонировать напрямую.

## Использование

```php
<?php

require __DIR__ . '/vendor/autoload.php';

$values = [23, 42, 8, 4, 15, 16];
$result = selectionSort($values);

// [4, 8, 15, 16, 23, 42]
```

Исходный массив не изменяется. Ключи нормализуются, повторяющиеся значения сохраняются.

## Поиск минимального элемента

`findMin()` возвращает индекс минимального значения после нормализации ключей массива.

```php
$index = findMin([23, 42, 8, 4, 15, 16]);

// 3
```

Для пустого массива функция выбрасывает `InvalidArgumentException`, поскольку минимального элемента не существует.

## Пользовательский порядок

`selectionSort()` и `findMin()` принимают callback-компаратор с тем же контрактом, что и `usort()`.

```php
$result = selectionSort(
	[3, 1, 4, 2],
	static fn (int $left, int $right): int => $right <=> $left,
);

// [4, 3, 2, 1]
```

### Сортировка объектов

```php
$items = [
	(object) ['name' => 'Gamma', 'priority' => 30],
	(object) ['name' => 'Alpha', 'priority' => 10],
	(object) ['name' => 'Beta', 'priority' => 20],
];

$result = selectionSort(
	$items,
	static fn (object $left, object $right): int => $left->priority <=> $right->priority,
);
```

## Сложность и свойства

Сортировка выбором выполняет `n × (n − 1) / 2` сравнений независимо от исходного порядка значений. Поэтому временная сложность в лучшем, среднем и худшем случаях — `O(n²)`.

Алгоритм выполняет не более `n − 1` обменов. Внутренняя логика использует `O(1)` вспомогательной памяти, но публичная функция возвращает новый массив и не изменяет переданный, поэтому PHP создаёт рабочую копию данных.

Классическая реализация с обменами не является стабильной: равные элементы могут изменить взаимный порядок. Ключи исходного массива не сохраняются.

## Тесты

```bash
composer install
composer test
```

Тесты проверяют пустой и одиночный массивы, дубликаты, отрицательные числа, строки, порядок ключей, пользовательский компаратор, объекты, неизменность входных данных и точное число сравнений.

## Benchmark

Диагностический benchmark показывает квадратичный рост времени и сравнивает учебную реализацию со встроенным `sort()`:

```bash
composer benchmark
```

Результаты зависят от версии PHP, процессора и текущей нагрузки. Benchmark не является критерием корректности и намеренно не запускается в CI.
